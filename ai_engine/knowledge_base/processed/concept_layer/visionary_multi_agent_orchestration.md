# 多智能体编排、审查与降级

## Supervisor 与协作

Supervisor 负责把目标转换为可执行的任务拆分，安排阶段顺序和角色协作，汇总 Specialist 的结果，并在质量检查失败时触发返修。它要记录任务状态和交接依据，使多智能体协作过程可以重放和审计，而不是只保留最终文本。

## Observe-Tool-Reflect-Revise 循环

observe-tool-reflect-revise 循环要求智能体先 observe 当前上下文和约束，再调用合适的 tool 获取真实结果，随后 reflect 工具输出、错误和证据缺口，最后 revise 原方案或答案。这个闭环能减少基于猜测继续执行的问题，并让失败原因与修订依据可审计。

## Critic 反馈与返修

CriticAgent 的职责是指出事实性、覆盖度、格式或证据问题。Specialist 应把这些反馈转换成明确的返修指令，必要时重新检索证据、重新调用工具并产生新版本；仅记录 Critic 反馈而不改变产物，不算完成审查闭环。

## 黑板模式的边界

Agent 黑板模式为多个角色提供共享的任务状态、证据和审计轨迹，便于异步协作和故障定位。黑板只是共享状态机制；如果任务分配和最终决策仍由 Supervisor 完成，系统依然属于中心控制，不应宣称为完全自治的分布式系统。

## 工具失败时的保守降级

工具调用失败后，智能体应记录失败原因和影响范围，重试可恢复错误，或改用已验证的可用证据生成保守结果。降级输出必须标注不确定性和缺失能力，不能把缺失证据伪装成事实，也不能用演示数据冒充真实调用结果。
