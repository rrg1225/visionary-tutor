# 人工智能全栈学习地图

## 为什么要按能力链学习

人工智能不是“模型名词的集合”，而是一条从问题定义到可信交付的能力链。学习时先判断任务属于预测、生成、检索、决策还是感知，再依次回答：数据从哪里来、目标函数是什么、如何训练、怎样评估、失败时怎样定位、上线后如何监控。只会调用模型而不能解释评估边界，不能算作完成学习闭环。

## 第一阶段：共同基础

先掌握 Python、线性代数、概率统计、微积分与优化，再学习监督学习、无监督学习、数据划分、过拟合、正则化和模型选择。初学者应能独立完成一个表格分类任务，并解释基线、损失函数、混淆矩阵、精确率、召回率和数据泄漏。Stanford CS229 的公开材料覆盖监督学习、生成式算法、学习理论、正则化、聚类、PCA 和强化学习，可作为理论索引；Google Machine Learning Crash Course 适合配合交互练习建立工程直觉。

## 第二阶段：方向分流

- 计算机视觉与多模态：卷积、视觉 Transformer、检测、分割、迁移学习、图文对齐和视觉问答。
- 自然语言处理与大模型：词向量、序列建模、注意力、Transformer、预训练、指令微调、检索增强生成和大模型评测。
- 强化学习与决策：马尔可夫决策过程、价值函数、策略梯度、Actor-Critic、离线强化学习和安全探索。
- 生成式人工智能：自回归生成、VAE、GAN、扩散模型、条件控制、事实性与安全评估。
- 数据与智能系统：特征工程、推荐、搜索、时序、图学习、MLOps、监控和负责任 AI。

## 第三阶段：项目证据

每个方向至少形成一份可复现证据包：问题定义、数据说明、训练配置、基线对比、消融实验、错误案例、限制与复现命令。高分项目不只展示成功案例，还要说明“不适用的输入”和“系统怎样拒绝不可靠结果”。

## 推荐学习顺序

1. 用传统机器学习建立完整评估闭环。
2. 用深度学习完成一个感知或语言任务，学会检查梯度、shape 和过拟合。
3. 选择一个主方向深入；本项目对计算机视觉保留更密集的测评、公式和调试案例。
4. 补齐 RAG、智能体、部署、监控、隐私与安全，形成端到端 AI 系统能力。
