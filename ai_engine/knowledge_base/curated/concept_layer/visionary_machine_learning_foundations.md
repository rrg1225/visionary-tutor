# 机器学习基础、泛化与评估

## 从任务到基线

机器学习首先是经验风险最小化问题：从有限样本学习一个能在未见数据上工作的函数。回归预测连续值，分类预测离散标签，聚类寻找没有显式标签的结构。开始训练前应写清输入、标签、业务代价和最简单基线；没有基线，复杂模型的收益无法判断。

## 数据划分与数据泄漏

训练集用于拟合参数，验证集用于调参和模型选择，测试集只用于最终评估。时间序列应按时间切分；同一用户或同一原始样本的派生数据不能跨集合，否则会产生数据泄漏。归一化、缺失值填充和特征选择只能在训练集上拟合，再应用到验证集和测试集。

## 损失、优化与正则化

均方误差常用于回归，交叉熵常用于分类。梯度下降沿损失下降方向更新参数，学习率过大会震荡或发散，过小会收敛缓慢。L1/L2 正则化、早停、数据增强和降低模型容量都能缓解过拟合，但应通过验证集曲线和误差分析判断，而不是机械叠加。

## 指标必须匹配代价

类别均衡且错误代价相近时可使用准确率；漏检代价高时关注召回率；误报代价高时关注精确率；类别不平衡时同时查看 PR-AUC、F1、混淆矩阵和分组指标。概率模型还应检查校准：置信度为 0.8 的样本是否大约有 80% 正确。单个总分可能掩盖少数群体或困难场景的失败。

## 误差分析清单

1. 先确认数据和标签是否正确。
2. 比较训练误差与验证误差，区分欠拟合和过拟合。
3. 按类别、来源、难度、时间和人群切片。
4. 记录代表性假阳性与假阴性，寻找系统模式。
5. 每次只改变一个关键变量并保留实验配置。
