# 计算机视觉与多模态进阶路线

## 从分类到结构化理解

图像分类为整张图预测标签；目标检测同时预测类别和位置；语义分割为每个像素预测类别；实例分割还要区分同类对象的不同实例。评测要与任务匹配：分类看混淆矩阵、F1 和校准，检测常看 IoU 与 mAP，分割常看 mIoU 和 Dice。小目标、遮挡、长尾类别和域偏移应单独切片。

## CNN 与视觉 Transformer

CNN 通过局部连接和权重共享建立空间归纳偏置，适合样本较少或计算受限场景。视觉 Transformer 把图像划分为 patch 并使用注意力建模全局关系，通常更依赖数据规模与预训练。选择架构时应比较精度、延迟、显存、数据量和部署平台，而不是只比较参数量。

## 多模态对齐

图文模型把视觉和语言映射到共享或可交互的表示空间。对比学习拉近匹配图文、推远不匹配样本；视觉问答和多模态生成还需要跨模态注意力或统一 token 表示。OCR 是多模态学习的重要边界：文字太小、压缩严重、反光、旋转或只有空白背景时，系统应先判断可读性，再决定识别和评测。

## 可靠视觉评测

输入门禁至少检查文件格式、字节大小、分辨率、解码结果和内容可读性。无 OCR 文本、极低置信度或演示占位图不得得到 0 分并写入画像；它们属于“无效样本”。应给用户可执行建议：保持光线均匀、文字占画面主体、避免反光、使用原图，并提供一个可下载的合格示例。

## 深入项目建议

在同一数据集上建立 CNN 基线，再加入迁移学习或 ViT；报告参数量、吞吐、各类别指标和失败案例。多模态项目还应做图像缺失、文本冲突、OCR 错误和提示注入测试。这样既保留 CV 的技术深度，也能与 NLP、RAG 和智能体形成真实系统闭环。
