# 生成式人工智能、RAG 与可信输出

## 生成模型家族

自回归模型把联合分布分解为一系列条件概率，适合文本、代码和离散序列生成。VAE 使用概率潜变量学习连续表示，训练目标包含重构项和分布正则项。GAN 通过生成器与判别器对抗训练，能产生清晰样本但可能模式崩溃。扩散模型逐步加噪并学习反向去噪，通常训练稳定、可控性强，但采样成本较高。

## RAG 的四段链路

1. 入库：清洗、语义切分、元数据和版本管理。
2. 检索：关键词与向量召回，必要时做查询改写。
3. 重排：把最相关、最可信的证据放到上下文前部。
4. 生成：只依据允许的证据回答，保留可核验引用。

检索相关不代表答案忠实。评测应拆成 Recall@k、MRR/nDCG、上下文精确率、答案正确性、忠实度与引用命中率。若没有证据、引用不存在、事实性得分低或多轮返修没有提升，资源必须进入降级或阻断状态，不能显示为“生成成功、引用通过”。

## 智能体的可靠执行

智能体把复杂任务拆成计划、工具调用、观察和修正。工具参数必须经过 schema 校验；会改变外部状态的操作要有权限边界和幂等设计；长任务要显示阶段、耗时和可恢复状态。反思或多智能体讨论不能替代独立的确定性检查，例如文件是否存在、引用 ID 是否来自本次检索、输出能否解析。

## 面向学习场景的拒答

教育产品的目标不是永远给出答案，而是给出可学习、可验证的帮助。输入不可读时应说明如何重拍；证据不足时给出已知边界和下一步；评测样本无效时不得写入学习画像。拒答、降级和人工复核都应是正式产品状态，而不是异常日志里的隐藏细节。
