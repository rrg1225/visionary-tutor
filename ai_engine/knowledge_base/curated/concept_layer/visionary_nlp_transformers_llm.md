# 自然语言处理、Transformer 与大模型

## 表示语言

自然语言处理把文本映射为可计算表示。分词器将文本转换为 token；词向量或上下文表示把 token 映射到连续空间。静态词向量对同一个词只有一个表示，上下文模型会根据句子环境改变表示，因此更能处理一词多义。

## 注意力与 Transformer

自注意力通过 Query 与 Key 的相似度计算权重，再对 Value 加权求和。缩放点积注意力可写为 `softmax(QK^T / sqrt(d_k))V`。多头注意力让模型在不同子空间捕捉句法、指代和长距离关系；位置编码补充序列顺序。训练时必须区分 padding mask 与 causal mask：前者忽略填充，后者防止生成模型看到未来 token。

## 大模型的训练与使用

自回归语言模型通过预测下一个 token 学习文本分布。预训练提供通用能力，指令微调改善任务遵循，偏好对齐改善输出行为，但这些步骤不会保证事实正确。推理时温度影响随机性，上下文窗口限制可见信息；长上下文也不等于模型一定能利用所有细节。

## 评测不能只看流畅度

语言任务应区分正确性、相关性、事实性、鲁棒性、安全性和延迟成本。自动指标适合大规模对比，但需要人工或独立 Judge 校准。对 RAG，应分别评估检索召回、证据相关性、答案忠实度和引用正确性；证据不足时明确拒答通常比流畅编造更好。

## 常见调试路径

- 输出重复：检查采样参数、停止条件和训练数据重复。
- 回答幻觉：检查检索证据、提示边界和引用门禁。
- 中文乱码或 Markdown 断裂：检查流式分块、字符编码与渲染规范化。
- 长文本遗漏：分层摘要、结构化检索，并对关键事实做位置扰动测试。
